Basic HTML Document Structure (DOCTYPE, html, head, body)
<!DOCTYPE> – declares HTML5 document type: This declaration tells the browser which version of HTML the document is written in, ensuring proper rendering and validation
html element – root element with lang attribute: The <html> element is tag is the root container for all HTML content; the lang attribute specifies the language for accessibility and SEO
head section – metadata, title, CSS links: Contains non-visible information like document title, character encoding, stylesheets, and meta tags for SEO
body section – visible content of the page: Contains all the content that users see and interact with in the browser
meta charset – specifies character encoding: Defines how characters are encoded, typically UTF-8 to support international characters and symbols
Semantic elements (header, nav, main, article, footer...)
header – logo, title, navigation: Defines introductory content at the top of a page or section, typically containing branding and primary navigation
nav – main navigation links: Contains primary navigation links for the site, improving accessibility and SEO structure
main – primary content (one per page): Represents the main content area of the document, unique to each page
article vs section - article vs section: article for self-contained content (like blog posts), section for thematic grouping of content
footer – copyright, links: Contains footer information like copyright, contact info, and secondary navigation
Headings , Paragraphs and other tags
h1 – main page title (most important): The primary heading that defines the main topic of the page, crucial for SEO and accessibility
h2-h6 – subheadings (hierarchical structure): Create a logical document outline with h2 as main sections, h3 as subsections, etc.
p element – paragraph text content: The standard element for blocks of text content, automatically creates line breaks
Paragraph spacing and line break ( <br> tag ); - : Browsers automatically add margin to paragraphs; use
for line breaks within paragraphs
Text formatting tags (strong, em, mark, etc..)
strong – important text (bold): Semantic element that indicates text of strong importance, typically rendered as bold
em – emphasized text (italic): Semantic element for emphasis, usually displayed as italic text, important for screen readers
mark – highlighted text: Used to highlight text for reference purposes, appears with a yellow background by default
small – fine print: Renders text in a smaller font size, commonly used for disclaimers and legal text
sup & sub – superscript and subscript: Used for mathematical expressions, footnotes, and chemical formulas
Image tag
src attribute – image URL: Specifies the path to the image file, can be relative or absolute URL
attribute – alternative text for image: Provides text description for accessibility and when images fail to load
width and height attributes Define image dimensions to prevent layout shifts and improve loading performance
images – srcset and sizes: Allows browsers to choose appropriate image based on screen size and resolution
Example of image tag – <img src="image.jpg" alt="My IMG">
Lists tag
ul – unordered list with bullet points: Creates lists with bullet points, ideal for non-sequential information like features or benefits
ol – ordered list with numbers: Creates numbered lists for sequential information like instructions or rankings
dl – description list with terms and definitions: Used for glossaries, FAQs, and key-value pairs with dt and dd elements
li – list item element: The container for each item within ul, ol, or dl lists
Nesting lists – lists within lists: Create hierarchical structures by placing lists inside other list items
Example of list tag – <ul><li>Fruits<ul><li>Apple</li><li>Banana</li></ul></li></ul>
Table tag
tr – table row cell: Defines a row inside a table
th – table header cell: Defines a heading cell in a table
td – table data cell: Defines a standard data cell
Example of table tag – <table><tr><th>Name</th><th>Age</th></tr><tr><td>Alex</td><td>22</td></tr></table>
Media tag
audio – audio container: Used to embed sound content in a web page
video – video container: Used to embed video content in a web page
source – media source: Specifies the media file and its format for audio or video
controls – playback controls: Adds play, pause, volume, and other controls
Example of audio tag – <audio controls><source src="song.mp3" type="audio/mpeg"></audio>
Example of video tag – <video width="320" height="240" controls><source src="movie.mp4" type="video/mp4"></video>
Progress , span and div tags
progress – progress indicator: Displays the progress of a task or operation
value – current progress value: Specifies the current progress amount
max – maximum value: Defines the total value needed to complete the progress
span – inline container: Used to style or group small portions of text
div – block-level container: Used to group and structure larger sections of content
Example of progress tag – <progress value="70" max="100"></progress>
Example of span tag – <p>This is a <span>highlighted</span> word</p>
Example of div tag – <div><p>Content inside a div</p></div>
Form tags
form – form container: Used to collect and submit user input
input – input field: Used to collect data from the user
type – input type: Specifies the kind of data the input accepts (text, email, password, etc.)
name – input name: Identifies form data when submitted
placeholder – hint text: Displays a short hint inside the input field
required – mandatory field: Makes the input field compulsory
button – form button: Used to submit or reset the form
Example of form tag – <form><label>Name</label><input type="text" required><button>Submit</button></form>
Ways to apply CSS (Inline, Internal, External)
Inline styles – style attribute on elements: Direct CSS applied to HTML elements using the style attribute, highest specificity but not reusable
Internal styles – <style> tag in <head> </head>: CSS rules placed within <style> tag in <head> </head>: CSS rules placed within <style> tag in <style> tags in the document head, scoped to that document
External stylesheets – separate .css files: External CSS files linked via separate CSS file: <link rel="stylesheet" href="styles.css">tag, promotes reusability and maintainability
Selectors in CSS
Element selector – selects HTML elements by tag name: p { color: red; }
Class selector – selects elements by class name: .box { background: blue; }
ID selector – selects an element by unique id: #header { font-size: 20px; }
Universal selector – selects all elements: * { margin: 0; padding: 0; }
Group selector – selects multiple elements: h1, h2, p { color: green; }
Box Model & box-sizing
Content – actual content area: The innermost area where text, images, and other content are displayed
Padding – space inside border: Creates space between content and border, affects background color/gradient
Border – edge around padding: The line that surrounds the padding and content area
Margin – space outside border: Creates space between elements, affects element positioning
box-sizing: - content-box vs border-box: content-box (default) adds padding/border to width/height, border-box includes them in specified width/height
Font, Text & Color Properties
font-family – sets the font type:Example: font-family: Arial, sans-serif;
font-size – controls text size:Example: font-size: 16px;
font-weight – controls text thickness:Example: font-weight: bold;
font-style – applies italic or normal style:Example: font-style: italic;
color – sets text color:Example: color: red;
background-color – sets background color:Example: background-color: lightblue;
text-align – aligns text horizontally:Example: text-align: center;
text-decoration – adds underline or line-through:Example: text-decoration: underline;
text-transform – changes letter case:Example: text-transform: uppercase;
line-height – controls space between lines:Example: line-height: 1.5;
var , let , const variables
var – function scoped, hoisted: Variables declared with var are function-scoped and hoisted to the top of their function, initialized with undefined
let – block scoped, can be reassigned: Variables declared with let are block-scoped and cannot be accessed before declaration (Temporal Dead Zone)
const – block scoped, cannot be reassigned: Constants declared with const are block-scoped and must be initialized at declaration
Examples for variables - var a = 10 ; let b = 20 ; const c = 30 ;
Functions (declaration, arrow)
Function declarations: – hoisted: Functions declared with function keyword are hoisted to the top of their scope and can be called before declaration
Function expressions: – not hoisted: Functions assigned to variables are not hoisted and must be defined before calling
Arrow functions: – no this binding: Arrow functions inherit this from enclosing scope, don't have their own this, arguments object, or prototype
Parameters and arguments: Parameters are function definition variables, arguments are the actual values passed when calling
Return statements and scope: Functions can return values and have access to variables in their lexical scope (closures)
Function Definition – a block of code that performs a task:Example:function greet() {console.log("Hello World");}
Function Call – runs the function:Example:greet();
Function with Parameter – accepts input values:Example:function add(a, b) {return a + b;}
Function Output – returns a result:Example:let result = add(5, 3);
JavaScript Operations
Arithmetic Operators – perform mathematical calculations:Example: let sum = 10 + 5;
Assignment Operators – assign values to variables:Example: let x = 10;
Comparison Operators – compare two values:Example: 5 == "5", 5 === "5";
Logical Operators – work with boolean values:Example: (x > 5 && y < 10);
Increment / Decrement – increase or decrease value by 1:Example: x++, y--;
Conditional (Ternary) Operator – short form of if-else:Example: let result = (age >= 18) ? "Adult" : "Minor";
Type Operators – check data type:Example: typeof 25;
JavaScript Array
Array Definition – an array stores multiple values in a single variable:Example:let numbers = [1, 2, 3];
Example 1: Access Element – access value using index:Example:numbers[0]; // 1
Example 2: Add Element – add value to array:Example:numbers.push(4);
Example 3: Loop Array – print all elements:Example:for (let i = 0; i < numbers.length; i++) {console.log(numbers[i]);}
Types of Arrays – different kinds of arrays:
• Single-Dimensional Array – let a = [1, 2, 3];
• Multi-Dimensional Array – let b = [[1, 2], [3, 4]];
JavaScript Conditional & Looping Statements
if statement – executes code when condition is true:Example:if (age >= 18) {console.log("Eligible to vote");}
if-else statement – chooses between two blocks:Example:if (marks >= 35) {console.log("Pass");} else {console.log("Fail");}
else-if ladder – checks multiple conditions:Example:if (score >= 90) {console.log("A Grade");} else if (score >= 60) {console.log("B Grade");} else {console.log("C Grade");}
switch statement – selects code based on value:Example:switch(day) {case 1: console.log("Monday"); break;case 2: console.log("Tuesday"); break;default: console.log("Invalid day");}
for loop – runs code a fixed number of times:Example:for (let i = 1; i <= 5; i++) {console.log(i);}
while loop – runs while condition is true:Example:let i = 1;while (i <= 5) {console.log(i);i++;}
do-while loop – executes at least once:Example:let j = 1;do {console.log(j);j++;} while (j <= 5);

Web Development Basics

Web browsers: Software applications that retrieve, render, and display web content (HTML, CSS, JavaScript).

Web servers: Systems that store and serve web pages and resources to browsers via HTTP/HTTPS.

MIME: Multipurpose Internet Mail Extensions; defines content types for files transmitted over the web.

URL: Uniform Resource Locator; a reference (address) to a resource on the internet.

HTTP: Hypertext Transfer Protocol; the foundation of data communication for the web.
